33      /**
34       * <p>
35       * New tokens must be added onto the end of the list with new numbers, and
36       * old tokens must remain and keep their current numbering. Old token
37       * numberings are not allowed to change.
38       * </p>
39       *
40       * <p>
41       * The reason behind this is Java inlines static final field values directly
42       * into the compiled Java code. This loses all connections with the original
43       * class, GeneratedJavaTokenTypes, and so numbering updates are not picked
44       * up in user-created checks and causes conflicts.
45       * </p>
46       *
47       * <p>
48       * Issue: https://github.com/checkstyle/checkstyle/issues/505
49       * </p>
50       */
