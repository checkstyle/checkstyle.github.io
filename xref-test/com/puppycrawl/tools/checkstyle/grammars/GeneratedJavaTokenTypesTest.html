31      /**
32       * <p>
33       * New tokens must be added onto the end of the list with new numbers, and
34       * old tokens must remain and keep their current numbering. Old token
35       * numberings are not allowed to change.
36       * </p>
37       *
38       * <p>
39       * The reason behind this is Java inlines static final field values directly
40       * into the compiled Java code. This loses all connections with the original
41       * class, GeneratedJavaTokenTypes, and so numbering updates are not picked
42       * up in user-created checks and causes conflicts.
43       * </p>
44       *
45       * <p>
46       * Issue: https://github.com/checkstyle/checkstyle/issues/505
47       * </p>
48       */
