036/**
037 * <p>
038 * The check finds classes that are designed for extension (subclass creation).
039 * </p>
040 * <p>
041 * Nothing wrong could be with founded classes.
042 * This check makes sense only for library projects (not application projects)
043 * which care of ideal OOP-design to make sure that class works in all cases even misusage.
044 * Even in library projects this check most likely will find classes that are designed for extension
045 * by somebody. User needs to use suppressions extensively to got a benefit from this check,
046 * and keep in suppressions all confirmed/known classes that are deigned for inheritance
047 * intentionally to let the check catch only new classes, and bring this to team/user attention.
048 * </p>
049 *
050 * <p>
051 * ATTENTION: Only user can decide whether a class is designed for extension or not.
052 * The check just shows all classes which are possibly designed for extension.
053 * If smth inappropriate is found please use suppression.
054 * </p>
055 *
056 * <p>
057 * ATTENTION: If the method which can be overridden in a subclass has a javadoc comment
058 * (a good practice is to explain its self-use of overridable methods) the check will not
059 * rise a violation. The violation can also be skipped if the method which can be overridden
060 * in a subclass has one or more annotations that are specified in ignoredAnnotations
061 * option. Note, that by default @Override annotation is not included in the
062 * ignoredAnnotations set as in a subclass the method which has the annotation can also be
063 * overridden in its subclass.
064 * </p>
065 * <p>
066 * Problem is described at "Effective Java, 2nd Edition by Joshua Bloch" book, chapter
067 * "Item 17: Design and document for inheritance or else prohibit it".
068 * </p>
069 * <p>
070 * Some quotes from book:
071 * </p>
072 * <blockquote>The class must document its self-use of overridable methods.
073 * By convention, a method that invokes overridable methods contains a description
074 * of these invocations at the end of its documentation comment. The description
075 * begins with the phrase “This implementation.”
076 * </blockquote>
077 * <blockquote>
078 * The best solution to this problem is to prohibit subclassing in classes that
079 * are not designed and documented to be safely subclassed.
080 * </blockquote>
081 * <blockquote>
082 * If a concrete class does not implement a standard interface, then you may
083 * inconvenience some programmers by prohibiting inheritance. If you feel that you
084 * must allow inheritance from such a class, one reasonable approach is to ensure
085 * that the class never invokes any of its overridable methods and to document this
086 * fact. In other words, eliminate the class’s self-use of overridable methods entirely.
087 * In doing so, you’ll create a class that is reasonably safe to subclass. Overriding a
088 * method will never affect the behavior of any other method.
089 * </blockquote>
090 * <p>
091 * The check finds classes that have overridable methods (public or protected methods
092 * that are non-static, not-final, non-abstract) and have non-empty implementation.
093 * </p>
094 * <p>
095 * Rationale: This library design style protects superclasses against being broken
096 * by subclasses. The downside is that subclasses are limited in their flexibility,
097 * in particular they cannot prevent execution of code in the superclass, but that
098 * also means that subclasses cannot corrupt the state of the superclass by forgetting
099 * to call the superclass's method.
100 * </p>
101 * <p>
102 * More specifically, it enforces a programming style where superclasses provide
103 * empty "hooks" that can be implemented by subclasses.
104 * </p>
105 * <p>
106 * Example of code that cause violation as it is designed for extension:
107 * </p>
108 * <pre>
109 * public abstract class Plant {
110 *   private String roots;
111 *   private String trunk;
112 *
113 *   protected void validate() {
114 *     if (roots == null) throw new IllegalArgumentException("No roots!");
115 *     if (trunk == null) throw new IllegalArgumentException("No trunk!");
116 *   }
117 *
118 *   public abstract void grow();
119 * }
120 *
121 * public class Tree extends Plant {
122 *   private List leaves;
123 *
124 *   &#64;Overrides
125 *   protected void validate() {
126 *     super.validate();
127 *     if (leaves == null) throw new IllegalArgumentException("No leaves!");
128 *   }
129 *
130 *   public void grow() {
131 *     validate();
132 *   }
133 * }
134 * </pre>
135 * <p>
136 * Example of code without violation:
137 * </p>
138 * <pre>
139 * public abstract class Plant {
140 *   private String roots;
141 *   private String trunk;
142 *
143 *   private void validate() {
144 *     if (roots == null) throw new IllegalArgumentException("No roots!");
145 *     if (trunk == null) throw new IllegalArgumentException("No trunk!");
146 *     validateEx();
147 *   }
148 *
149 *   protected void validateEx() { }
150 *
151 *   public abstract void grow();
152 * }
153 * </pre>
154 * <ul>
155 * <li>
156 * Property {@code ignoredAnnotations} - Specify annotations which allow the check to
157 * skip the method from validation.
158 * Default value is {@code After, AfterClass, Before, BeforeClass, Test}.
159 * </li>
160 * </ul>
161 * <p>
162 * To configure the check:
163 * </p>
164 * <pre>
165 * &lt;module name=&quot;DesignForExtension&quot;/&gt;
166 * </pre>
167 * <p>
168 * To configure the check to allow methods which have @Override and @Test annotations
169 * to be designed for extension.
170 * </p>
171 * <pre>
172 * &lt;module name=&quot;DesignForExtension&quot;&gt;
173 *   &lt;property name=&quot;ignoredAnnotations&quot; value=&quot;Override, Test&quot;/&gt;
174 * &lt;/module&gt;
175 * </pre>
176 * <pre>
177 * public class A extends B {
178 *   &#64;Override
179 *   public int foo() {
180 *     return 2;
181 *   }
182 *
183 *   public int foo2() {return 8;} // violation
184 * }
185 *
186 * public class B {
187 *   &#47;**
188 *    * This implementation ...
189 *      &#64;return some int value.
190 *    *&#47;
191 *   public int foo() {
192 *     return 1;
193 *   }
194 *
195 *   public int foo3() {return 3;} // violation
196 * }
197 *
198 * public class FooTest {
199 *   &#64;Test
200 *   public void testFoo() {
201 *     final B b = new A();
202 *     assertEquals(2, b.foo());
203 *   }
204 *
205 *   public int foo4() {return 4;} // violation
206 * }
207 *</pre>
208 *
209 * @since 3.1
210 */
211@StatelessCheck
212public class DesignForExtensionCheck extends AbstractCheck {
213
214    /**
215     * A key is pointing to the warning message text in "messages.properties"
216     * file.
217     */
218    public static final String MSG_KEY = "design.forExtension";
219
220    /**
221     * Specify annotations which allow the check to skip the method from validation.
222     */
223    private Set<String> ignoredAnnotations = Arrays.stream(new String[] {"Test", "Before", "After",
224        "BeforeClass", "AfterClass", }).collect(Collectors.toSet());
225
226    /**
227     * Setter to specify annotations which allow the check to skip the method from validation.
228     * @param ignoredAnnotations method annotations.
229     */
230    public void setIgnoredAnnotations(String... ignoredAnnotations) {
231        this.ignoredAnnotations = Arrays.stream(ignoredAnnotations).collect(Collectors.toSet());
232    }
233
234    @Override
235    public int[] getDefaultTokens() {
236        return getRequiredTokens();
237    }
238
239    @Override
240    public int[] getAcceptableTokens() {
241        return getRequiredTokens();
242    }
243
244    @Override
245    public int[] getRequiredTokens() {
246        // The check does not subscribe to CLASS_DEF token as now it is stateless. If the check
247        // subscribes to CLASS_DEF token it will become stateful, since we need to have additional
248        // stack to hold CLASS_DEF tokens.
249        return new int[] {TokenTypes.METHOD_DEF};
250    }
251
252    @Override
253    public boolean isCommentNodesRequired() {
254        return true;
255    }
256
257    @Override
258    public void visitToken(DetailAST ast) {
259        if (!hasJavadocComment(ast)
260                && canBeOverridden(ast)
261                && (isNativeMethod(ast)
262                    || !hasEmptyImplementation(ast))
263                && !hasIgnoredAnnotation(ast, ignoredAnnotations)) {
264            final DetailAST classDef = getNearestClassOrEnumDefinition(ast);
265            if (canBeSubclassed(classDef)) {
266                final String className = classDef.findFirstToken(TokenTypes.IDENT).getText();
267                final String methodName = ast.findFirstToken(TokenTypes.IDENT).getText();
268                log(ast, MSG_KEY, className, methodName);
269            }
270        }
271    }
272
273    /**
274     * Checks whether a method has a javadoc comment.
275     * @param methodDef method definition token.
276     * @return true if a method has a javadoc comment.
277     */
278    private static boolean hasJavadocComment(DetailAST methodDef) {
279        return hasJavadocCommentOnToken(methodDef, TokenTypes.MODIFIERS)
280                || hasJavadocCommentOnToken(methodDef, TokenTypes.TYPE);
281    }
282
283    /**
284     * Checks whether a token has a javadoc comment.
285     *
286     * @param methodDef method definition token.
287     * @param tokenType token type.
288     * @return true if a token has a javadoc comment.
289     */
290    private static boolean hasJavadocCommentOnToken(DetailAST methodDef, int tokenType) {
291        final DetailAST token = methodDef.findFirstToken(tokenType);
292        return branchContains(token, TokenTypes.BLOCK_COMMENT_BEGIN);
293    }
294
295    /**
296     * Checks whether a specified token type exist under token.
297     *
298     * @param token tree token.
299     * @param tokenType token type.
300     * @return true if a tokenType exists under token.
301     */
302    private static boolean branchContains(DetailAST token, int tokenType) {
303        boolean result = false;
304        DetailAST curNode = token;
305        while (curNode != null) {
306            if (curNode.getType() == tokenType) {
307                result = true;
308                break;
309            }
310
311            DetailAST toVisit = curNode.getFirstChild();
312            while (toVisit == null) {
313                if (curNode == token) {
314                    break;
315                }
316
317                toVisit = curNode.getNextSibling();
318                curNode = curNode.getParent();
319            }
320            curNode = toVisit;
321        }
322
323        return result;
324    }
325
326    /**
327     * Checks whether a methods is native.
328     * @param ast method definition token.
329     * @return true if a methods is native.
330     */
331    private static boolean isNativeMethod(DetailAST ast) {
332        final DetailAST mods = ast.findFirstToken(TokenTypes.MODIFIERS);
333        return mods.findFirstToken(TokenTypes.LITERAL_NATIVE) != null;
334    }
335
336    /**
337     * Checks whether a method has only comments in the body (has an empty implementation).
338     * Method is OK if its implementation is empty.
339     * @param ast method definition token.
340     * @return true if a method has only comments in the body.
341     */
342    private static boolean hasEmptyImplementation(DetailAST ast) {
343        boolean hasEmptyBody = true;
344        final DetailAST methodImplOpenBrace = ast.findFirstToken(TokenTypes.SLIST);
345        final DetailAST methodImplCloseBrace = methodImplOpenBrace.getLastChild();
346        final Predicate<DetailAST> predicate = currentNode -> {
347            return currentNode != methodImplCloseBrace
348                && !TokenUtil.isCommentType(currentNode.getType());
349        };
350        final Optional<DetailAST> methodBody =
351            TokenUtil.findFirstTokenByPredicate(methodImplOpenBrace, predicate);
352        if (methodBody.isPresent()) {
353            hasEmptyBody = false;
354        }
355        return hasEmptyBody;
356    }
357
358    /**
359     * Checks whether a method can be overridden.
360     * Method can be overridden if it is not private, abstract, final or static.
361     * Note that the check has nothing to do for interfaces.
362     * @param methodDef method definition token.
363     * @return true if a method can be overridden in a subclass.
364     */
365    private static boolean canBeOverridden(DetailAST methodDef) {
366        final DetailAST modifiers = methodDef.findFirstToken(TokenTypes.MODIFIERS);
367        return ScopeUtil.getSurroundingScope(methodDef).isIn(Scope.PROTECTED)
368            && !ScopeUtil.isInInterfaceOrAnnotationBlock(methodDef)
369            && modifiers.findFirstToken(TokenTypes.LITERAL_PRIVATE) == null
370            && modifiers.findFirstToken(TokenTypes.ABSTRACT) == null
371            && modifiers.findFirstToken(TokenTypes.FINAL) == null
372            && modifiers.findFirstToken(TokenTypes.LITERAL_STATIC) == null;
373    }
374
375    /**
376     * Checks whether a method has any of ignored annotations.
377     * @param methodDef method definition token.
378     * @param annotations a set of ignored annotations.
379     * @return true if a method has any of ignored annotations.
380     */
381    private static boolean hasIgnoredAnnotation(DetailAST methodDef, Set<String> annotations) {
382        final DetailAST modifiers = methodDef.findFirstToken(TokenTypes.MODIFIERS);
383        final Optional<DetailAST> annotation = TokenUtil.findFirstTokenByPredicate(modifiers,
384            currentToken -> {
385                return currentToken.getType() == TokenTypes.ANNOTATION
386                    && annotations.contains(getAnnotationName(currentToken));
387            });
388        return annotation.isPresent();
389    }
390
391    /**
392     * Gets the name of the annotation.
393     * @param annotation to get name of.
394     * @return the name of the annotation.
395     */
396    private static String getAnnotationName(DetailAST annotation) {
397        final DetailAST dotAst = annotation.findFirstToken(TokenTypes.DOT);
398        final String name;
399        if (dotAst == null) {
400            name = annotation.findFirstToken(TokenTypes.IDENT).getText();
401        }
402        else {
403            name = dotAst.findFirstToken(TokenTypes.IDENT).getText();
404        }
405        return name;
406    }
407
408    /**
409     * Returns CLASS_DEF or ENUM_DEF token which is the nearest to the given ast node.
410     * Searches the tree towards the root until it finds a CLASS_DEF or ENUM_DEF node.
411     * @param ast the start node for searching.
412     * @return the CLASS_DEF or ENUM_DEF token.
413     */
414    private static DetailAST getNearestClassOrEnumDefinition(DetailAST ast) {
415        DetailAST searchAST = ast;
416        while (searchAST.getType() != TokenTypes.CLASS_DEF
417               && searchAST.getType() != TokenTypes.ENUM_DEF) {
418            searchAST = searchAST.getParent();
419        }
420        return searchAST;
421    }
422
423    /**
424     * Checks if the given class (given CLASS_DEF node) can be subclassed.
425     * @param classDef class definition token.
426     * @return true if the containing class can be subclassed.
427     */
428    private static boolean canBeSubclassed(DetailAST classDef) {
429        final DetailAST modifiers = classDef.findFirstToken(TokenTypes.MODIFIERS);
430        return classDef.getType() != TokenTypes.ENUM_DEF
431            && modifiers.findFirstToken(TokenTypes.FINAL) == null
432            && hasDefaultOrExplicitNonPrivateCtor(classDef);
433    }
434
435    /**
436     * Checks whether a class has default or explicit non-private constructor.
437     * @param classDef class ast token.
438     * @return true if a class has default or explicit non-private constructor.
439     */
440    private static boolean hasDefaultOrExplicitNonPrivateCtor(DetailAST classDef) {
441        // check if subclassing is prevented by having only private ctors
442        final DetailAST objBlock = classDef.findFirstToken(TokenTypes.OBJBLOCK);
443
444        boolean hasDefaultConstructor = true;
445        boolean hasExplicitNonPrivateCtor = false;
446
447        DetailAST candidate = objBlock.getFirstChild();
448
449        while (candidate != null) {
450            if (candidate.getType() == TokenTypes.CTOR_DEF) {
451                hasDefaultConstructor = false;
452
453                final DetailAST ctorMods =
454                        candidate.findFirstToken(TokenTypes.MODIFIERS);
455                if (ctorMods.findFirstToken(TokenTypes.LITERAL_PRIVATE) == null) {
456                    hasExplicitNonPrivateCtor = true;
457                    break;
458                }
459            }
460            candidate = candidate.getNextSibling();
461        }
462
463        return hasDefaultConstructor || hasExplicitNonPrivateCtor;
464    }
